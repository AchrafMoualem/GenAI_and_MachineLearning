# Object-Oriented Programming Concepts

## What is a class?
A **class** is like a **blueprint** or recipe for creating things. Just like a cookie cutter shapes cookies, a class defines what something should look like and do.

## What is an instance?
An **instance** is the **actual thing** you create from the blueprint. If class is the cookie cutter, the instance is the cookie you make with it.

## What is encapsulation?
**Encapsulation** means **keeping related stuff together** and hiding the messy details inside. 

## What is abstraction?
**Abstraction** means **hiding complex stuff** and showing only what's needed. Like driving a car - you use the steering wheel, not the engine parts.

## What is inheritance?
**Inheritance** is when a **child class gets features from a parent class**. Like kids inheriting traits from parents.

## What is multiple inheritance?
**Multiple inheritance** is when a **child inherits from multiple parents**. Like a kid getting traits from both mom and dad.

## What is polymorphism?
**Polymorphism** means **many forms** - the same method name working differently for different classes. Like "play" means different things for a dog vs a cat.

## What is method resolution order or MRO?
**MRO** is Python's **way of deciding which parent's method to use** when there's multiple inheritance. Like: "Who do I ask first - mom or dad?"
